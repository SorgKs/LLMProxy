# План функции `_detect_full_function_presence`

## Цель
Определить, присутствует ли в уже собранном (возможно частичном) содержимом файла полный исходный код функции с заданным именем. Это необходимо перед конвертацией `function_replace` → `apply_diff` для решения, достаточно ли текущих данных или нужно запросить недостающие части файла через `read_file`.

## Входные параметры
- **file_lines** – список строк файла, полученный из `_extract_file_content_from_request`. Индекс = номер строки ‑ 1. Пустые строки (`""`) означают, что соответствующая часть файла неизвестна.
- **function_name** – строка, имя функции, которую нужно найти.
- **language** – (опционально) язык файла (`python`, `javascript` и т.д.). На данный момент реализована поддержка только Python, но структура оставлена открытой для расширения.

## Алгоритм (словесное описание)

1. **Поиск объявления функции**
   - Проходим `file_lines` последовательно.
   - Ищем строку, которая после удаления ведущих пробелов начинается с `def <function_name>` и заканчивается двоеточием `:`.
   - При нахождении фиксируем её индекс как `start_idx`.

2. **Сбор комментария ПЕРЕД функцией**
   - Начиная со строки `start_idx - 1` двигаемся вверх.
   - Добавляем подряд идущие строки-комментарии/пробелы в отдельный блок «комментарий перед функцией».
   - Как только встречаем непробельную строку, которая не является комментарием, останавливаемся.

3. **Проверка полноты кода и границ**
   - Сразу после нахождения начала функции проходим по строкам:
     * **Отсутствует строка** (в `file_lines` нет данного индекса) → сразу возвращаем ошибку: код функции неполный (нужно запрашивать файл).
     * **Строка с отступом <= исходному отступу объявления** (и это не первая строка тела) → тело закончилось; возвращаем полный код функции и номер строки начала.
   - При этом дополнительно собираем «комментарий ПЕРЕД функцией» (см. п. 2).
   - Если все строки до конца тела заполнены и до следующей функции/объекта отступ не уменьшился — возвращаем полный код функции и собранный комментарий ПЕРЕД функцией.

4. **Возврат результата**
   - Функция возвращает кортеж `(has_full_code: bool, start_line: int | None, end_line: int | None, leading_comment: str | None)`. При `has_full_code=False` значения `start_line`/`end_line` могут быть `None`.

## Как будет использоваться
- В `process_single_tool_call` перед попыткой конвертации `function_replace` вызываем `_detect_full_function_presence`.
- Если `has_full_code = True` → сразу формируем `apply_diff`.
- Если `False` → генерируем `read_file`‑tool‑call, запрашивая недостающий диапазон строк (можно запросить весь файл или только диапазон, где обнаружены пустые строки).

## Расширяемость
- Для других языков (JS/TS) вместо поиска `def` будет использоваться соответствующий паттерн (`function <name>` или `const <name> = (`) и аналогичный анализ отступов/скобок.
- При необходимости можно добавить параметр `max_lines` для ограничения количества запрашиваемых строк.
